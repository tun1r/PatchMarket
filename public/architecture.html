System map

Four moving parts, one payout rule.

Buyer opens the market. PatchMarket gates the claim. Worker submits the patch. Verifier decides whether escrow unlocks.

Protocol path

HTTP first, money only on the gated path.

The buyer never gets a worker claim for free. Claim is blocked behind L402, then Agent CAPTCHA, then proof-driven verification.

Truth in demo

What is real, what is staged for reliability.

The architecture video should be honest. Judges trust systems that clearly separate production mechanics from demo scaffolding.

Real today
  • Buyer agent runs real tool calls against the API
  • HTTP claim path returns real 402 Payment Required
  • Agent CAPTCHA issues and validates a real cryptographic challenge
  • Verifier shells out to Node in a temp worktree
  • Exit codes, logs, proof hash, and workspace apply are real
Demo reliability choices
  • Lightning settlement is simulated/regtest style
  • Worker patch is deterministic by default for stage safety
  • Market depth is mocked to the two workers needed for the wedge
Release rule

Escrow unlocks only on verified green.

That is the whole product. Agents can buy verified outcomes instead of raw model tokens, raw completions, or untrusted code patches.
